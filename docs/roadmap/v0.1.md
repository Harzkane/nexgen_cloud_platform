# NCP Milestone Roadmap: v0.1 — Foundation

The goal of Version 0.1 is to establish a secure, production-grade OS foundation on an Ubuntu server. Rather than creating a single installer script, we are splitting Version 0.1 into iterative releases to ensure our core plugin architecture is fully tested and proven before installing any software.

---

## 🗺️ v0.1 Release Stages

### 🔍 v0.1-alpha — Module Discovery & Validation
**Objective:** Build the platform's core discovery system. Confirm NCP can inspect its own modules, read metadata, and check target operating system specs without changing any system configurations.

- **Command:** `ncp doctor`
- **Deliverables:**
  - **Module Discovery Engine (`core/loader/`):** Recursively scans the `modules/` folder for subdirectories containing `metadata.yml`.
  - **Metadata Parser:** Parses YAML files to validate name, version, priorities, OS support, and dependencies.
  - **Validator Engine (`core/validator/`):** Runs environment tests (checks target OS version, verifies network connection, checks user permissions).
  - **Dry-Run Planner:** Outputs the module discovery map and lists the planned execution order based on priority.

---

### 🏗️ v0.1-beta — Installer Engine & Logging
**Objective:** Build the orchestration pipeline. Implement the idempotency rules, hooks, and logging capabilities that future modules will depend on.

- **Command:** `ncp install --dry-run` and `ncp install --module <name>`
- **Deliverables:**
  - **Installer Engine (`core/installer/`):** Coordinates sequential module execution and handles dependencies.
  - **Execution Hooks:** Implements module hook standards (`install`, `verify`, `status`).
  - **Logging Engine (`core/logger/`):** Saves structured logs to `/var/log/ncp/` or local `logs/` and outputs neat progress bars/results.

---

### 🚀 v0.1.0 — Foundation Release
**Objective:** Deliver the production-grade Ubuntu base system. Build the first set of functional modules executing actual system configurations.

- **Command:** `ncp install`
- **Target Modules to Execute:**
  - **system/git**: Installs and validates Git.
  - **system/curl**: Installs and validates Curl.
  - **system/build-tools**: Installs base compilers.
  - **system/docker**: Configures Docker Engine and Compose.
  - **security/firewall**: Sets up default UFW rules (allows 22, 80, 443).
  - **security/fail2ban**: Configures brute-force SSH blocking.
  - **security/ssh**: Disables root password logins, enforces SSH keys.
- **Deliverable:** Fully hardened host Ubuntu OS with a generated installation status report.

---

## 🛠️ Verification Criteria

### v0.1-alpha Verification
- Running `ncp doctor` successfully prints the categories (System, Databases, Security, Monitoring, Backup, CI, Networking, Runtime), lists the active `.yml` files, and prints a target dependency graph.

### v0.1.0 Verification
- Verifying the final server setup against the standard checklist:
  - Password auth disabled in `/etc/ssh/sshd_config`.
  - Firewall active with open ports `22`, `80`, `443` only.
  - Fail2ban service running and jails active.
  - Docker engine and compose running.
  - Host directory `/opt/nexgen/` created and correctly permissions-bound.
